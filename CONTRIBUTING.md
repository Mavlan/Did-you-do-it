# Contribution Guidelines

Thank you for your interest in contributing to the Did-you-do-it project!

## How to Contribute
1. **Fork the repo**: Start by forking the repository to your own GitHub account.
2. **Clone your fork**: Clone the forked repository to your local machine using `git clone <your-fork-url>`.
3. **Create a new branch**: Before making changes, create a new branch for your work with `git checkout -b feature/your-feature-name`.
4. **Make your changes**: Make sure to write clear, concise commit messages that describe your changes.
5. **Test your changes**: Ensure that your changes don’t break any existing functionality.
6. **Push to your fork**: Push your changes to your forked repository with `git push origin feature/your-feature-name`.
7. **Open a Pull Request**: Go to the original repository and open a pull request against the `main` branch. Add a description of your changes.

## Code Style
- Follow the coding conventions used in the project.
- Maintain a clean and organized code structure.

## Reporting Issues
If you find a bug, please open an issue in this repository. Provide as much detail as possible to help us understand the problem.

## Questions?
Feel free to reach out to the maintainers for any queries about the contribution process!
